# Clawworld AI 美术流水线

日期：`2026-03-22`

## 结论

`Clawworld` 的游戏本体继续使用：

- `Phaser`：场景、角色、碰撞、动画、状态机
- `React`：抽屉、面板、白板详情、异常卡
- `Vite`：开发与构建

AI 出图工具不负责“做游戏”，只负责“做素材”。

最稳的分工是：

- `Gemini Ultra`：美术总监，负责概念图、风格统一、角色/家具/房间语言
- `Nano Banana`：可选素材车间，负责立绘、宣传图、单张角色或物件概念图
- `Codex + Phaser 代码`：把概念图收口成真正可运行的游戏图块、动画和交互

一句话：

**`Gemini` 决定画什么，`Nano Banana` 帮忙产出单张素材，`Phaser` 决定这些东西怎么活起来。**

## 为什么不是“全靠 Nano Banana”

`Nano Banana` 更像插画师，不像整条游戏生产线。

它适合做：

- 角色肖像
- 宣传图
- 单张办公室家具概念图
- 同一角色的情绪变化参考图

它不适合直接当最终游戏素材总引擎的原因：

- 很难稳定产出同一套像素角色的多方向、多帧动作
- 很难保证一整套办公室家具都严格共用同一个比例和网格
- 很难直接满足 `Phaser` 需要的 `sprite sheet`、`atlas`、透明边距和命名规范

也就是说，`Nano Banana` 适合“画脸”和“画海报”，不适合直接“出整套零件箱”。

## 正式分工

### 1. Gemini Ultra：总美术

负责：

- 主大厅、任务世界、后勤区的场景定调
- 角色原型设计
- 家具轮廓与功能识别语言
- 色板、材质、灯光方向
- 每一轮重构前的视觉方向文档

产出形式：

- 场景方向文档
- 角色/家具 sprite 指南
- 概念图提示词
- “哪些字该消失、哪些功能改成物件”的整改清单

当前已经有的 Gemini 文档：

- `docs/2026-03-22-office-art-direction.md`
- `docs/2026-03-22-task-world-art-direction.md`
- `docs/2026-03-22-sprite-direction.md`

### 2. Nano Banana：可选素材车间

负责：

- 单角色立绘
- 宣传页用视觉图
- 某个家具的单张风格草图
- 已有像素角色的“非游戏内”放大形象图

暂不负责：

- 直接产出最终可用的多帧走路动画
- 直接产出一整套办公室 tile set
- 直接产出最终可用的交互式 UI sprite atlas

使用原则：

- 先有 `Gemini` 的风格文档，再让 `Nano Banana` 出图
- 只接“单张图任务”，不接“整套系统任务”
- 所有出图都要能回到同一色板、同一比例、同一世界观

### 3. Codex / Phaser：收口与落地

负责：

- 把概念图翻译成真正可运行的像素物件
- 建立角色、家具、房间的网格尺寸
- 建立交互热点、层级、阴影、状态动画
- 决定什么是运行时对象，什么只是视觉参考

简单说：

**AI 负责想象和画参考，代码负责让它变成一套能拼、能动、能测试的零件。**

## 推荐资产类型

第一阶段优先做这几类：

### A. 游戏内必须资产

- 角色小图块：24x24 或 32x32
- 办公室家具图块：白板、任务板、工位、财务室门、工具柜、档案柜
- 状态帧：亮屏、灭屏、抽屉开、告警灯亮、纸条贴上
- 地面和墙面 tile

这些必须最终收口为：

- 固定像素尺寸
- 固定透明边界
- 固定命名
- 可在 `Phaser` 中直接调用

### B. 游戏外辅助资产

- 角色大头像
- 宣传图
- 任务世界封面图
- 公司大屏展示图

这些更适合让 `Nano Banana` 发挥。

## 当前正式路线

按优先级排序：

1. `Gemini Ultra` 定风格和下一轮视觉方向
2. `Codex` 先用代码把轮廓、碰撞区、房间语言做对
3. 等轮廓稳定后，再决定要不要批量引入 `Nano Banana` 做大图和部分概念图
4. 真正的游戏内 sprite sheet，仍然由 `Phaser` 这边按统一网格收口

## 不该做的事

- 不要让 `Nano Banana` 直接决定整套游戏风格
- 不要先出很多漂亮大图，再回头硬塞进游戏
- 不要混用多套不同色板和比例
- 不要把宣传图生产线当成游戏素材生产线

## 下一步

下一轮美术和场景重构，继续按这条线执行：

1. `Gemini Ultra` 继续给主大厅和任务世界出更细的像素资产方向
2. 代码侧把“代码画出来的物件”逐步替换为更接近真实 sprite 的结构
3. 当角色、白板、工具柜、财务室门这批轮廓彻底稳定后，再决定是否接 `Nano Banana` 做立绘与宣传资产
